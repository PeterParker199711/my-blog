### 🔍 嫌疑人二：Vite 的“超强缓存”在作祟

Vite 在处理引入的原始文件（`?raw` 或 `.md` 文件）时，有时候为了速度会进行极强的缓存。你改了 `.md` 文件，但浏览器里加载的可能还是几分钟前的旧内容！

**✅ 排查方法：**

1.  在终端里按 `Ctrl + C` 停掉当前运行的服务。
2.  重新运行 `npm run dev`。
3.  回到浏览器，**按 `Ctrl + F5` 强制刷新页面**。

-----

### 🔍 嫌疑人三：点击了复制，但是没反应？

如果你看到了“复制”按钮，但是点击它没变成蓝色的“已复制！”，可能是因为浏览器的安全策略。
现代浏览器的 `navigator.clipboard.writeText` API **只能在安全上下文（Secure Context）中运行**。

  * 如果你是在 `http://localhost:5173` 访问，它是允许的。
  * 如果你是在手机上通过局域网 IP（比如 `http://192.168.1.100:5173`）访问，部分浏览器可能会静默拦截剪贴板的写入！

-----
